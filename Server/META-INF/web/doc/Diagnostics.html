Brief description:

The diagnostics page displays information about the running Scheduling Server. This is to help diagnose potential problems and investigate error conditions. The diagnostics displayed on this page is automatically updated every 10 seconds.

Runtime information:

The information shown on this tab is about the execution of the Scheduling Server. The information fields are:
  • Name - The runtime name of the Java virtual machine. The exact name format is virtual machine specific but is usually composed of <Process PID>@<Computer hostname>.
  • Start time - The time when the Scheduling Server was started.
  • Uptime - How long the Scheduling Server has been running in terms of days, hours, minutes and seconds.
  • Working directory - The current / working directory (CWD) of the Scheduling Server which is the base path of relative file paths.
    NOTE: The working directory is extremely important as the configuration files are loaded relative to it.
    HINT: The Scheduling Server service sets the working directory to the installation directory of the Scheduling Server.
  • Class path - The list of directories and Jar files where classes are loaded from. Any class path location starting with '.' is relative to the working directory of the Scheduling Server.
  • Boot class path - The list of directories and Jar files where the Java classes are loaded.
  • Library path: The list of directories where native libraries (i.e. for use with JNI) are loaded from.
  • System properties - The system properties of the Scheduling Server. System properties are used to configure the Scheduling Server configuration system (i.e. configuration files).
  • Environment variables - The environment variables given to the Scheduling Server by the invocation environment.

System

The information shown on this tab displays memory being used by the Scheduling Server and active threads.
  • In use memory - Shows the amount of memory current in use by the Scheduling Server in megabytes.
  • Free memory - Free memory shows the amount of memory that is assigned to the Scheduling Server and not being used.
  • Total memory - Total memory shows the memory assigned to the Scheduling Server by the operating system.
  • Max memory - Max memory shows the amount of memory that can be assigned to the Scheduling Server by the operating system if it is needed.
  • Current threads - Shows the number of currently active threads.
  • Daemon threads - Daemon threads do not keep the Java virtual machine running (i.e. when all threads are daemon threads, the Scheduling Server terminates).
  • Peak threads - The highest number of concurrent threads that have been running.
  • Total started threads - The total number of tests that have been started.
  • Thread stack traces - These list of active threads. The information shown about thread is:
    • Thread unique identifier.
    • Thread name.
    • Thread state.
    • Thread CPU time, rather how long the thread has been in the runnable state.
    • Thread stack trace. This shows the currently executing method at the top of the stack and the list of method invocations that from thread start got to the current method.

Java

The information show on this tab displays Java virtual machine details.
  • Virtual machine - The name of the virtual machine.
    HINT: Java usually comes with a few virtual machine implementations including 'Client' and 'Server' (the virtual machine name will specifies this). It is recommended to use the 'Server' virtual machine which trades memory and startup time for better performance. This may be configured in the Scheduling Server service INI file by setting the path of the virtual machine implementation library.
  • version - The Java virtual machine version.
    NOTE:The minimum required version is '1.6.x', also known as Java 6 (this might be displayed as 16.x). Earlier versions of Java are unsupported and probably will not work.
  • Vendor - The Java virtual machine vendor.
  • Specification - The virtual machine specification.
  • JIT compilation - Whether the virtual machine support just-in-time compilation.
  • JIT compiler - The name of the JIT compiler.
  • Compilation time - The amount of time the virtual machine has spent compiling byte code to native code.
  • Number of garbage collectors - The number of garbage collectors in use.
  • Garbage collectors - The name of the garbage collectors.
  • Garbage collections - The number of times garbage collection has run.
  • Garbage collection CPU time - The cumulative time taken to run garbage collections.
  • Loaded classes - The number of loaded classes.
  • Unloaded classes - The number of unloaded classes.
  • Total loaded classes - The total number of loaded classes.

OS

The information show on this tab display host operating system details.
  • Operating system - The name of the operating system.
  • Version - The operating system version.
  • Arch - The operating system architecture.
    NOTE: On Linux 64bit architectures (displayed as 'amd64') should use a Scheduling Server service executable compiled for 64bit Linux.
  • Processors - The number of processors available to the Java virtual machine.
  • File separator - The separator between directories in file paths. This is dependent on the operating system running.
    HINT: The separtor '/' works on both UNIX and Windows and should be used Windows instead of the displayed character '\'. The '\' character is used as the special character escape character, so if '\' is used, it should be escaped using '\' (i.e. C:\\foo\\bar\\baz).
  • Path separator - The character delimiting file paths in the Scheduling Servers class path and library path. This is dependant on the operating system running.
  • File system roots - The list of top most directories in the filesystem.
  • File system <x> - The free and total space on the file system <x>.
  • Temp directory - The operating system temporary directory. The Scheduling Server by default stores uploaded batch files in this directory.
